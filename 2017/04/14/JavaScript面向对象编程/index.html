JavaScript面向对象编程

  • JavaScript 面向对象

一、面向对象和面向过程

【面向过程思想】
处理某个问题，看的是“如何”处理这个问题，是一种数学逻辑的映射。

【面向对象思想】
处理某个问题，看的是“谁”来处理这个问题，把问题拆解成各个“对象（谁）”，“对象”作为接收消息的单位，“对象”处理相应的功能逻辑，“对象”之间相互协同完成工作，是一种生活逻辑的映射。

【面向对象相关术语】
1、类：一种事物的抽象，创建对象的模板。
2、对象：某种类的一个实例（真实的，唯一的产物）。
3、属性：是对象的特征。
4、方法：对象的行为。

【从封装的角度分析面向对象】
函数：封装功能逻辑。
类：封装了属性（特征）和方法（行为）

面向对象编程，我们首先得有真实的对象去处理问题，对象是通过类去创建出来的，但是JS中没有“类”（ES6已经支持类的概念了）的概念。所以我们在面向对象编程处理问题之前，先设计出来需要用到的“类”，设计完后，我们需要去模仿“类”的功能。

二、对象字面量

对象：是一种特殊的数据类型，可以存储任何结构的数据。而且具有对数据的封装性。
字面量：简化声明的一种方式。比如之前学习的声明一个字符串：var str = “abc”; 数组：var arr = [“a”, “b”, “c”] 都是字面量语法。

三、设计模式

1.工厂模式：所谓工厂就是一个函数，根据参数生成对象，就像工作的车间一样，批量生产。

  • 工厂模式存在创建的对象不能区分类型的问题

2.构造函数模式：本质上就是一个函数，只不过函数的名字首字母我们一般大写，为了区分普通函数。构造函数可以模拟类的概念。通过构造函数就可以创建出来我们想要的类型的对象。

  • 构造函数存在方法得不到公用，每个新的实例都会创建一个新的方法，浪费内存的问题。

3.构造函数+原型模式
原型：每个构造函数都有一个原型属性（prototype），这个原型属性会指向一个原型对象，我们可以给这个原型对象动态的设置属性和方法，这些属性和方法可以给该构造函数创建出来的对象实例所使用。而且达到了属性和方法共享的功能。

  • 通过原型添加的方法可以使方法得到公用。以后的每个实例共享在原型添加的方法。
  • 构造函数使用new和不使用new的区别

四、 原型链

每个对象都有一个对应的原型对象，这个原型对象本身也是一个对象，那么既然它是对象，所以它也有相应的原型对象，以此类推，一直到Object.prototype为止，Object.prototype的原型对象是null，通过以上分析，会形成一条链子，这条链就是原型链。

原型链的一个非常重要的特性：当使用某个对象的某个属性或者方法的时候，首先检查当前对象是否有这个属性或方法，如果有就用，如果没有就沿着原型链往上找，找到当前对象对应的原型对象，对应的原型对象如果有则可以拿来使用，如果没有则继续通过原型链往上找，直到找不到为止，就没有这个属性或方法了。
